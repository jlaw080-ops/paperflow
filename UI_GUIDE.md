# UI 디자인 가이드

## 디자인 원칙
1. **종이처럼 읽힌다.** 기술 보고서가 주 콘텐츠다. 화면에서도, 인쇄에서도 본문 가독성이 최우선.
2. **장식보다 여백.** 강조색·아이콘·그림자를 절제하고 충분한 여백과 행간으로 정돈한다.
3. **화면과 인쇄의 일관성.** 화면에서 보던 글이 인쇄해도 같은 느낌이어야 한다.
4. **도구는 뒤로, 글은 앞으로.** 사이드바·버튼 등 UI는 조용히, 본문이 주인공.

## AI 슬롭 안티패턴 — 하지 마라
| 금지 사항 | 이유 |
|-----------|------|
| `backdrop-filter: blur()` (글래스모피즘) | AI 템플릿의 가장 흔한 징후 |
| 그라데이션 텍스트(gradient-text) | AI가 만든 SaaS 랜딩의 1번 특징 |
| 네온 글로우 box-shadow 애니메이션 | 네온 글로우 = AI 슬롭 |
| 보라/인디고 브랜드 색상 | "AI = 보라색" 클리셰 |
| 모든 카드에 동일한 `rounded-2xl` | 균일한 둥근 모서리는 템플릿 느낌 |
| 배경 그라데이션 orb 장식 | 모든 AI 랜딩 페이지에 있는 장식 |

## 색상 (종이 톤)
- 배경: 거의 흰색 `#FCFCFB` (순백 `#FFFFFF`보다 약간 따뜻하게)
- 본문 텍스트: 진회색 `#1F2328` (순검정 `#000`은 피함 — 눈이 피로)
- 보조 텍스트/메타: `#6B7280`
- 경계선: `#E5E7EB`
- 강조색(링크·선택 등): 절제된 단색 1개. 예) 청록 계열 `#0F766E` 또는 남색 `#1D4ED8` 중 택1.
- 시맨틱: 경고 `#B45309`, 위험 `#B91C1C` — 꼭 필요할 때만.
- **인쇄 시**: 본문은 검정에 가깝게, 배경은 흰색. 강조색은 인쇄에서도 과하지 않게.

## 타이포그래피
- 본문 폰트: **Pretendard**(한글·영문 가독성이 좋은 한국어 웹 표준급 산세리프) 권장. 없으면 시스템 산세리프 폴백.
- 코드/표 숫자: 고정폭(monospace) — `ui-monospace`, `D2Coding` 등.
- 본문 크기: 16~17px, 행간 1.7 정도(읽기 편하게).
- 제목 위계(h1~h3)를 크기·굵기로 분명히. h1은 문서 제목 한 번만.

## 레이아웃
- 편집 화면: 좌측 **폴더 트리(고정 너비 ~280px)** + 우측 편집/미리보기 영역.
- 공개 보기: 가운데 정렬, 본문 폭 `max-width: 760px` 내외(긴 줄은 읽기 어렵다).
- 인쇄: 본문 폭 제한 해제, A4 여백 20mm. (상세는 `ARCHITECTURE.md`의 print.css)

## 컴포넌트
- **폴더 트리**: 들여쓰기로 계층 표현, 폴더는 펼침/접힘. 선택된 항목만 옅은 배경으로 표시(강한 색 금지).
- **버튼**: 기본은 테두리 있는 차분한 버튼. 주요 동작 1개만 채운 버튼.
- **카드**: 꼭 필요할 때만. 모서리는 `rounded-md` 정도로 절제, 그림자는 아주 옅게(또는 테두리로 대체).

## 애니메이션 (허용 목록 — 이 외 금지)
- 트리 펼침/접힘: 짧은 높이 전환(150ms 내외).
- 버튼/링크 hover: 색 전환 정도.
- 그 외 등장 애니메이션, 글로우, 패럴랙스 등은 쓰지 않는다.
